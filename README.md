# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

・リモートリポジトリ
ネットワーク上やファイル上に存在する複数の利用者で共有するリポジトリの事。
・ローカルリポジトリ
ユーザーがローカルマシン上で作業するために利用するリポジトリの事。

## プッシュとマージの違いは何でしょうか？
プッシュはローカルからリモートへ編集内容を反映させ、
マージはローカルからリモートとまたがず、編集内容を編集すること。

## コミットとプッシュの違い
・コミット
ゲームでいうセーブのようなもの。
仮に何かのバグ等でデータが消えてしまっても、最後にコミットした場所からやり直すことができる。
・プッシュ
ローカルからリモートへ編集内容を反映させるもの。
コミットは作業にあたって、都度編集内容を保存していくもので、
その編集した内容をリモートへ反映させるかの違い。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
チームで管理を行うことが多いので、
ログを見返したときに、誰がどこの指摘に対してどのような編集を行なったのかを
わかるようにすることが重要。
※各プロジェクトやチーム内でのルールも定められている部分もあるのでそれに従う。


## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
レビューの依頼をするかしないかの違い。
コードレビューをせずに反映をしてしまうとどこか編集内容に不備があることに気づけないので、
プルリクエストをつくり、レビューをしてらってから問題がないようであればそのままマージすることができる。


## コンフリクトを起こしてしまった場合、どう対処すべきですか？
対処の方法は3つの編集パターンがある。
①先にマージされた変更内容を取り込む。
②後にマージしようとしている変更内容を取り込む。
③どちらの変更内容も取り込む。
